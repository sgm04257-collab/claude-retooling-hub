강의 핵심 12가지

Anthropic Academy 공식 강의 21강의 핵심 내용을 아코디언으로 정리했습니다. 클릭해서 펼쳐보세요.

① 시작하기 — 설치 및 초기화▼
Node.js 18+ 필요.
npm install -g @anthropic-ai/claude-code cd project && /init
초기 스캔 및 스타터 메모리 파일 생성. Claude Code = AI 현장 직원. tool use로 파일 읽기·수정·명령어 실행.
② CLAUDE.md — AI 업무 매뉴얼▼
대상: Tech stack, Architecture
이유: Purpose, Design decisions
방법: Build/test commands, Workflows, Gotchas

매 세션 시작 시 자동 로드. Git에 커밋해서 팀 공유.
③ 메모리 파일 3계층▼
~/.claude/CLAUDE.md # 글로벌 (전체) ./CLAUDE.md # 프로젝트 (Git 공유) ./frontend/CLAUDE.md # 서브폴더 (스코프)
각 파일 200줄 미만 · 절대 상위 컨텍스트 덮어쓰지 않음.
④ 모범 사례▼
먼저 /init 후 출력 다듬기 · 구체적인 지시사항 · @filename 문서 참조
Claude가 유추 못하는 유의사항 추가 · 메모리는 간결하게 · 팀 공유 위해 Git 커밋
⑤ 프로젝트 파일 구조▼
your-project/ ├── CLAUDE.md ├── .claude/ │ ├── settings.json │ ├── skills/ │ │ ├── code-review/SKILL.md │ │ └── testing/SKILL.md │ ├── commands/deploy.md │ └── agents/security-reviewer.md └── src/
⑥ 스킬 추가 — 자연어 호출▼
--- name: testing patterns description: jest testing patterns — 언제 쓸지로 작성 allowed_tools: Read, Grep, Glob --- # Testing Patterns Use describe+it+AAA pattern Use factory mocks
설명 필드는 "언제 이 스킬을 실행할지"로 작성. 자동 활성화에 핵심.
⑦ AI 엔지니어 스킬 아이디어▼
① 코드 리뷰 (보안·XSS 자동 감지)
② 테스트 패턴 (TDD 자동화)
③ 커밋 메시지 (Conventional Commits)
④ 도커 배포 스킬
⑤ 코드베이스 시각화
⑥ API 디자인 가이드
⑧ Hooks — 자동 안전 가드레일▼
"hooks": { "PreToolUse": [{ "matcher": "Bash", "hooks": [{ "type": "command", "command": "scripts/sec.sh", "timeout": 5 }] }] }
종료코드 0 → 허용, 1+ → 차단
⑨ 권한 및 안전▼
"permissions": { "allow": ["Read:*","Bash:git:*","Write:*:*.md"], "deny": ["Read:env:*","Bash:sudo:*"] }
⑩ 4단계 아키텍처▼
L1 – CLAUDE.md: 지속 컨텍스트 및 규칙
L2 – Skills: 자동 호출 지식 팩
L3 – Hooks: 안전 게이트 및 자동화
L4 – Agents: 자체 컨텍스트 하위 에이전트
⑪ 일일 워크플로우 패턴▼
cd project && /init Shift+Tab × 2 → 플랜 모드 Shift+Tab → 자동 수락 모드 /compact → 컨텍스트 압축 Esc Esc → 되감기 자주 커밋 · 특징당 새 세션 시작
⑫ 빠른 참조 명령어▼

12가지 핵심 설정법

잘 쓰는 사람과 못 쓰는 사람의 차이는 CLAUDE.md 하나. 세션이 끊겨도 규칙이 살아있으려면 여기에 기록해야 합니다.

파일 위치 & 범위
관리 정책 (강제)
/etc/claude-code/CLAUDE.md
조직 전체 강제 적용 · 사용자 오버라이드 불가. macOS: ~/Library/Application Support/ClaudeCode/CLAUDE.md
프로젝트 (팀 공유)
./CLAUDE.md
프로젝트 아키텍처 · 코딩 표준 · Git으로 팀 공유 필수. 모노레포에서 가장 중요.
사용자 (개인)
~/.claude/CLAUDE.md
개인 코드 스타일 · 선호도. Git에 올리지 않는 개인 설정.
❌ vs ✅ 지침 작성법
❌ Claude가 무시합니다
"코드를 제대로 포맷합니다"
"변경사항을 테스트합니다"
"API를 src/api에 위치시킵니다"
✅ 검증 가능한 지시
"2칸 들여쓰기 사용"
"커밋 전 npm test 실행"
"API는 src/api/handler에 위치"
핵심 규칙 12가지
④
200줄 이하 유지
길수록 준수율 하락. 많아지면 @import나 .claude/rules/로 분리.
⑤
@import로 분리
@README @package.json @~/.claude/my-prefs.md — Git 싫은 개인 설정은 홈 디렉토리에서 로드.
⑥
.claude/rules/ 분리
code-style.md / testing.md / security.md — 주제별 분리 → 팀 유지관리 쉬워짐.
⑦
경로별 규칙
paths: "src/api/**/*.ts"
해당 파일 열 때만 규칙 로드. 컨텍스트 절약.
⑧
자동 메모리 구분
CLAUDE.md = 내가 씀
Auto Memory = Claude가 씀
"항상 pnpm 써" → 자동 기억.
⑨
로드 순서
관리정책 → 사용자 → 상위 → 프로젝트 → 하위
claudeMdExcludes로 다른 팀 파일 제외.
⑩
/memory 습관화
현재 세션 로드 파일 확인. 목록에 없으면 Claude는 못 봄. 쓰고 나서 반드시 확인.
⑪
/compact 후 소실 주의
대화에만 있던 지침은 compact 후 사라짐. CLAUDE.md에 기록해야 영구 유지.
⑫
핵심 요약
CLAUDE.md = 구체적·간결·200줄↓
Auto Memory = 자동 축적
.claude/rules/ = 대규모
/memory = 로드 확인 필수
🛠 Skills 꿀팁

Anthropic 내부 9가지 유형 + 핵심 원칙

Thariq Shihipar (Claude Code 팀)이 수백 개 스킬 운영하며 얻은 교훈.

9가지 스킬 유형
1
API 레퍼런스
라이브러리 사용 시 흔한 실수·주의사항 사전 명시.
예: axios 타임아웃 기본값 함정
2
제품 검증
가입·결제 같은 핵심 흐름을 헤드리스 브라우저로 자동 테스트.
예: Playwright E2E 결제 플로우
3
데이터 수집·분석
모니터링 대시보드 연결, 코호트 비교, 퍼널 분석 자동화.
예: Mixpanel → 전환율 리포트
4
반복 업무 자동화
스탠드업·티켓 생성·주간 리캡을 한 명령으로.
예: "스탠드업 작성해" → Jira 자동
5
코드 템플릿
새 기능 만들 때 팀 규칙에 맞는 뼈대 자동 생성.
예: React 컴포넌트 팀 컨벤션 적용
6
코드 품질·리뷰
서브에이전트가 코드를 비판하고 수정을 반복.
예: 보안 리뷰어 XSS 자동 검사
7
CI/CD & 배포
PR 감시·테스트 재시도·점진적 롤아웃·자동 롤백.
예: 배포 실패 → 자동 롤백 + Slack
8
장애 대응 런북
에러 증상 → 관련 로그 수집 → 보고서 자동 작성.
예: "500 에러 급증" → 원인 리포트
9
인프라 운영
고아 리소스 탐색·의존성 관리·비용 급증 원인 분석.
예: AWS 미사용 리소스 → 비용 절감
핵심 원칙 4가지
⚠️
① 함정 목록(Pitfall List)을 반드시 넣을 것 — 가장 중요
AI가 자주 틀리는 부분을 미리 적어두면 같은 실수를 반복하지 않습니다. 새로운 실패가 발견될 때마다 한 줄씩 추가하는 것이 핵심입니다.
## Pitfalls - axios 타임아웃 기본값은 0 (무한대) — 반드시 timeout: 5000 명시 - React key는 index 대신 고유 ID 사용 - 2025-01 이후 버전은 deprecated API 사용 금지
📂
② 점진적 공개(Progressive Disclosure) — 파일로 나눌 것
상세한 API 시그니처는 references/api.md에, 출력 템플릿은 assets/ 폴더에 분리. 모든 정보를 한 번에 넣으면 AI가 오히려 혼란스러워합니다.
my-skill/ ├── SKILL.md ← 핵심 지침만 ├── references/api.md ← 상세 API └── assets/template.md ← 출력 템플릿
🎯
③ 너무 세세하게 지시하지 말 것
핵심만 알려주고 상황 판단은 AI에게 맡겨야 합니다. 모든 경우의 수를 통제하려 하면 오히려 유연성이 떨어집니다.
📝
④ Description 필드는 AI를 위해 쓸 것
"이 스킬이 뭘 하는가"가 아니라 "언제 이 스킬을 실행할지"로 써야 합니다.

❌ "코드 리뷰를 수행합니다" (기능 설명)
✅ "사용자가 'PR 리뷰해줘', '코드 검토', '보안 점검'을 요청할 때 실행" (트리거 설명)
핵심 통찰
하나의 카테고리만
좋은 스킬은 하나의 카테고리에 명확히 속합니다. "이 스킬이 뭘 하는 거지?"라는 질문이 나오면 범위가 너무 넓다는 신호.
몇 줄로 시작해도 됩니다
완벽한 스킬을 설계하려 하기보다 시작하고 반복하는 것이 핵심. 처음엔 몇 줄 메모 + 함정 하나로 시작.
실패할 때마다 한 줄
AI가 새로운 실수를 할 때마다 함정 목록에 한 줄 추가. 이것이 스킬을 점점 좋게 만드는 유일한 방법.
🧠 메모리 & Auto Dream

기억력 10배 올리기

Claude Code 오래 쓸수록 기억력이 떨어진다고 느끼셨다면? Auto Dream 설정 하나로 달라집니다.

메모리 3종류
CLAUDE.md
내가 쓰는 지침
규칙·컨텍스트·제약. 매 세션 자동 로드. 200줄 이하 권장.
Auto Memory
Claude가 쓰는 노트
작업하며 자동 기록. "항상 pnpm 써" → 기억. 세션 간 자동 축적.
Auto Dream
메모리 자동 정리
중복 삭제·분류. 모순 내용 정리. 쓸모없는 메모 버림. 날짜 정확히 변환.
📌 포스트잇 범벅 비유: 신입사원이 포스트잇에 뭐든 적으면 한 달 후 모니터가 범벅입니다. 같은 내용 3장씩, 언제 적었는지 모르는 메모들... Auto Dream을 켜면 자면서 기억을 정리합니다. 잠을 못 자면 기억력이 떨어지는 것처럼, Claude도 정리 없이는 점점 멍청해집니다.
Auto Dream 설정법 3단계
1
Claude Code에서 /memory 입력
현재 세션에 로드된 모든 파일 목록 확인. Auto Memory 설정 패널이 나타납니다.
2
